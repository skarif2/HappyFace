I create responsive wevsites that allow the user to experience your website in the best and most appropriate way suited to the device they are using. By using progressive enhancement, your website is delivered with a responsive layour that can best make use of the space available on the smallest to largest devices. Cross browser compatibility is ensured by using feature detection so older browesers still in use today provide a solid experience, while modern browsers can go the next step and enhace a user's time spent on your site.


Writing semantic markup that is documented and wasy to read means it can be maintained and scaled in the future, and allows co-workers to quickly work with it.


Rather that create website on a page-by-page basis, I insted consider each element of your screen-based designs as seperate components that can exist elsewere on a website outside of the contest of your design compositions. With these componenets I produce a style guid that acts as a document for a reusable and maintainable code base. All inbolbed in a project can rely on a style guide to understand a project's language and produce a more consistent user experience.


My front-end build process involes the use of tools such as Sass, Gulp, Bower nad NPM to speed up development.

I can qickly and efficiently join your team using continuous-integragion with mehtods and tools such as Git, Jenkins, Agile/Scrum and test/behaviour diven development.
